## Part 2：内存管理（★★★★★）

### Stack vs Heap

| | Stack | Heap |
|---|-------|------|
| 分配 | 编译期/进入作用域自动 | `new` / `malloc` 运行时 |
| 释放 | 离开作用域自动 | 手动 `delete` / `free` |
| 速度 | 极快（移动栈指针） | 较慢（分配器、碎片） |
| 大小 | 有限（通常 MB 级） | 大（受虚拟内存限制） |
| 碎片 | 无 | 可能 |

### new vs malloc

| | `new` / `new[]` | `malloc` |
|---|-----------------|----------|
| 类型 | 类型安全，调用构造 | 只分配原始字节 |
| 失败 | 抛 `std::bad_alloc` | 返回 `nullptr` |
| 大小 | `sizeof(T)` 自动 | 手动计算字节 |
| 释放 | `delete` / `delete[]` | `free` |
| 重载 | 可重载 `operator new` | 不可（除非替换 malloc） |

```cpp
// new：分配 + 构造
auto* p = new Foo(42);

// malloc：只分配
void* raw = std::malloc(sizeof(Foo));
Foo* p2 = static_cast<Foo*>(raw);
new (p2) Foo(42);   // placement new：在已有内存上构造
```

### delete vs free

- `delete`：调用析构 + 释放内存
- `free`：只释放内存，**不调用析构** → 对 C++ 对象用 `free` 是 UB

### placement new

```cpp
alignas(Foo) char buf[sizeof(Foo)];
Foo* p = new (buf) Foo(42);   // 在 buf 上构造，不分配堆
p->~Foo();                     // 必须手动析构；buf 内存自己管理
```

**用途：** 内存池、嵌入式固定地址、STL allocator、`vector` 未初始化内存上构造元素。

### operator new / delete

```cpp
void* operator new(std::size_t sz) {
    void* p = std::malloc(sz);
    if (!p) throw std::bad_alloc();
    return p;
}
```

可全局替换分配策略（jemalloc、tcmalloc、追踪泄漏）。

### 常见陷阱

| 陷阱 | 后果 |
|------|------|
| Memory Leak | `new` 后未 `delete` |
| Dangling Pointer | 释放后仍使用指针 |
| Double Free | 同一块内存 `delete` 两次 |
| `delete` vs `delete[]` | `new[]` 必须用 `delete[]`，否则 UB |

**Q: 为什么 delete[] 和 delete 不一样？**

`new[]` 可能在分配块前存元素个数；`delete[]` 对每个元素调析构并正确释放整块。`delete` 只析构一个对象。

## Part 3：指针（★★★★★）

### const 修饰谁？（读法：从名字往外读）

```cpp
int* p;           // 指向 int 的指针
const int* p;     // 指向 const int（*p 不可改）
int* const p;     // const 指针（p 不可改）
const int* const p; // 都不可改
```

```cpp
int x = 1, y = 2;
const int* p1 = &x;
// *p1 = 3;     // ❌
p1 = &y;          // ✅ 指针本身可变

int* const p2 = &x;
*p2 = 3;          // ✅
// p2 = &y;     // ❌
```

详见 [`examples/03_const_pointers.cpp`](../examples/03_const_pointers.cpp)。

### 其他要点

| 概念 | 说明 |
|------|------|
| **Pointer Arithmetic** | `p+1` 移动 `sizeof(T)` 字节 |
| **void*** | 通用指针，**不知道类型不能解引用**（不知 size/alignment） |
| **nullptr** | 类型为 `std::nullptr_t`，可隐式转为任意指针；**比 NULL（0）更安全**，避免 `f(int)` 与 `f(int*)` 歧义 |
| **Wild Pointer** | 未初始化指针 |
| **Dangling Pointer** | 指向已释放内存 |

**Q: 为什么 void* 不能直接解引用？**

编译器不知道指向多少字节、什么对齐；必须 `static_cast<T*>` 后再用。

## Part 9：内存布局（★★★★★）

### 经典面试题

```cpp
class A {
    char c;   // 1 byte
    int i;    // 4 bytes
};
// sizeof(A) == ?  通常是 8（不是 5）
```

**原因：对齐（Alignment）**

- `int` 通常要求 4 字节对齐
- `char c` 占 offset 0；`int i` 必须从 offset 4 开始
- offset 4–7 放 `i`；总大小 8，尾部无浪费

```cpp
class B {
    int i;    // offset 0, 4 bytes
    char c;   // offset 4, 1 byte
};
// sizeof(B) == 8（尾部 padding 3 bytes 使整体对齐到 4）
```

### 工具

```cpp
sizeof(A)                    // 总大小
alignof(int)                 // 对齐要求，通常 4
offsetof(A, i)               // 成员偏移
```

### 规则

1. 成员按声明顺序排列，offset 递增
2. 每个成员的 offset 必须是 `alignof(成员)` 的倍数
3. 结构体总大小是最大成员对齐的倍数

### 虚函数对布局的影响

有虚函数时，对象**开头**通常有 hidden **vptr**（8 bytes on 64-bit），然后才是成员。

详见 [`examples/08_memory_layout.cpp`](../examples/08_memory_layout.cpp)。

### Cache 关联（Part 15 预告）

结构体成员重排可减少 padding、提高 cache 利用率：

```cpp
// 差：24+ bytes
struct Bad { char a; double b; char c; };

// 好：16 bytes
struct Good { double b; char a; char c; };
```

False sharing：多线程写同一 cache line 不同变量 → 见 [06-并发与内存模型.md](./06-并发与内存模型.md)。
