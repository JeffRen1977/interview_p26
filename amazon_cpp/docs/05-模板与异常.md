## Part 11：Templates（★★★★☆）

### Function Template

```cpp
template <typename T>
T max_val(const T& a, const T& b) {
    return (a < b) ? b : a;
}
// 编译器为每个使用的 T 生成一份代码（实例化）
```

### Class Template

```cpp
template <typename T>
class Stack {
    std::vector<T> data_;
public:
    void push(const T& x) { data_.push_back(x); }
    T pop() { T t = data_.back(); data_.pop_back(); return t; }
};
```

### Template Specialization

```cpp
template <typename T> struct IsPointer { static const bool value = false; };
template <typename T> struct IsPointer<T*> { static const bool value = true; };
```

### Variadic Template

```cpp
template <typename... Args>
void log(Args&&... args) {
    (std::cout << ... << args) << '\n';  // C++17 fold expression
}
```

### SFINAE（Substitution Failure Is Not An Error）

替换失败时从重载决议中移除，不报错：

```cpp
template <typename T>
typename std::enable_if<std::is_integral<T>::value, T>::type
foo(T x) { return x + 1; }

template <typename T>
typename std::enable_if<std::is_floating_point<T>::value, T>::type
foo(T x) { return x + 0.5; }
```

C++20 用 **Concept** 更清晰：

```cpp
template <std::integral T>
T foo(T x) { return x + 1; }
```

**面试深度：** Senior 岗知道 SFINAE/Concept 即可；Principal 可能问 `std::enable_if` vs `requires`、模板实例化对编译时间和二进制体积的影响。

## Part 12：异常（★★★★☆）

### 基本语法

```cpp
try {
    risky();
} catch (const std::exception& e) {
    log(e.what());
} catch (...) {
    log("unknown");
}
```

### noexcept

```cpp
void safe() noexcept;  // 承诺不抛异常；若抛出 → std::terminate
```

移动构造/赋值标 `noexcept` 时，`vector` 扩容才用移动而非拷贝。

### 为什么析构函数不要抛异常？

1. 析构可能在栈展开时因另一异常被调用
2. C++ 标准：若析构抛异常且没有 active exception → `std::terminate`
3. 两个异常同时传播无法安全处理

**规则：析构函数 swallow 或 noexcept；错误用错误码/状态标志在析构前处理。**

### 异常安全等级

| 等级 | 保证 |
|------|------|
| **Basic** | 不泄漏资源；对象仍有效 |
| **Strong** | 失败时状态不变（copy-and-swap） |
| **No-throw** | 绝不失败 |

```cpp
Buffer& operator=(const Buffer& o) {
    Buffer tmp(o);  // strong：tmp 构造失败则 *this 不变
    swap(tmp);
    return *this;
}
```

### Amazon Infra 实务

很多底层 C++ 代码（驱动、热路径）**禁用异常**（`-fno-exceptions`），用 `Status` / `Expected<T,E>` 模式。面试时可以说两种风格都熟悉。
