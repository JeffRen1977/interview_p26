## Part 13：并发（★★★★★）

### 同步原语一览

| 类型 | 用途 |
|------|------|
| `mutex` | 互斥锁 |
| `recursive_mutex` | 同一线程可多次 lock（递归调用） |
| `shared_mutex` | 读多写少：shared_lock 读 / unique_lock 写 |
| `lock_guard` | RAII 锁，构造 lock、析构 unlock |
| `unique_lock` | 可 defer、try、与 condition_variable 配合 |
| `condition_variable` | 等待条件，配合 mutex |
| `atomic<T>` | 无锁原子操作 |
| `future` / `promise` | 异步结果传递 |
| `async` | 启动异步任务 |

### mutex vs spinlock

| | mutex | spinlock |
|---|-------|----------|
| 等待 | 阻塞，让出 CPU | 忙等循环 |
| 适用 | 临界区较长、竞争一般 | 临界区极短、核绑定、不愿上下文切换 |
| 风险 | 优先级反转（可用 priority inheritance） | 浪费 CPU、长时间自旋性能差 |

### condition_variable 标准模式

```cpp
std::mutex mtx;
std::condition_variable cv;
std::queue<int> q;
bool done = false;

// consumer
std::unique_lock lock(mtx);
while (!done || !q.empty()) {          // while 防虚假唤醒
    cv.wait(lock, [&]{ return done || !q.empty(); });
    if (!q.empty()) { auto x = q.front(); q.pop(); }
}

// producer
{ std::lock_guard lock(mtx); q.push(42); }
cv.notify_one();
```

手撕版：[`interview_handwrite/cpp/bounded_blocking_queue.cpp`](../interview_handwrite/cpp/bounded_blocking_queue.cpp)

### atomic 为什么快？

简单读写在支持 CPU 上用 **LOCK-free 原子指令**（如 x86 `LOCK CMPXCHG`），无需进入内核态 mutex。

### CAS（Compare-And-Swap）

```cpp
bool compare_exchange_weak(T& expected, T desired);
// 若 *this == expected，则 *this = desired，返回 true
// 否则 expected = *this，返回 false
```

无锁数据结构（栈、队列、引用计数）的基础。

### ABA Problem

线程 A 读 ptr=P，被挂起；B 把 P 改成 Q 又改回 P；A 的 CAS 仍成功但结构已变。

**解决：** 带版本号的指针（tagged pointer）、Hazard Pointer、Epoch-based reclamation。

## Part 14：Memory Model（★★★★☆）

### 两种 Reorder

| | 谁重排 | 目的 |
|---|--------|------|
| **Compiler Reorder** | 编译器 | 优化 |
| **CPU Reorder** | 处理器 | 流水线、Store Buffer |

多线程下重排可能导致数据竞争；用 `atomic` + `memory_order` 约束。

### memory_order

| 顺序 | 语义 | 场景 |
|------|------|------|
| `relaxed` | 原子性，无同步 | 纯计数器 |
| `acquire` | 读：此后操作不重排到此前 | consumer 读 flag |
| `release` | 写：此前操作不重排到此后 | producer 写 flag |
| `acq_rel` | 读+写 | read-modify-write |
| `seq_cst` | 全局顺序（默认） | 最简单，略慢 |

**经典发布-订阅：**

```cpp
std::atomic<bool> ready{false};
int data = 0;

// thread 1 (producer)
data = 42;
ready.store(true, std::memory_order_release);

// thread 2 (consumer)
while (!ready.load(std::memory_order_acquire)) {}
assert(data == 42);  // 保证看到 42
```

SPSC ring buffer 见 [`interview_handwrite/cpp/spsc_ring_buffer.cpp`](../interview_handwrite/cpp/spsc_ring_buffer.cpp)。

## Part 15：Cache（★★★★☆）

### 基本概念

| 概念 | 说明 |
|------|------|
| **Cache Line** | 通常 64 bytes；CPU 按 line 加载 |
| **Spatial Locality** | 访问相邻地址 → 预取命中 |
| **Temporal Locality** | 近期访问的数据易在 cache |
| **False Sharing** | 两线程写**同一 cache line 不同变量** → line 来回失效 |

### False Sharing 示例与修复

```cpp
// 差：counter1 和 counter2 可能在同一 cache line
struct Bad {
    std::atomic<int> counter1;
    std::atomic<int> counter2;
};

// 好：padding 到不同 cache line
struct alignas(64) Good {
    std::atomic<int> counter1;
    char pad[64 - sizeof(std::atomic<int>)];
    std::atomic<int> counter2;
};
```

**Amazon Infra 场景：** per-core 统计、metrics 计数器、多生产者队列的 head/tail 分离（SPSC 无锁队列）。

### vector 为什么快？

连续内存 → cache friendly；`unordered_map` 节点分散 → cache miss 多。
