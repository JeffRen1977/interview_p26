## Virtual 机制

```cpp
class Base {
public:
    virtual void foo() { std::cout << "Base\n"; }
    virtual ~Base() = default;
    int x = 0;
};

class Derived : public Base {
public:
    void foo() override { std::cout << "Derived\n"; }
    int y = 0;
};
```

### 内存布局

```
Derived 对象:
┌──────────┐
│ vptr ────┼──→ Derived vtable
├──────────┤     ┌─────────────────┐
│ Base::x  │     │ &Derived::foo   │
├──────────┤     │ &Derived::~Derived│
│ Derived::y     │ typeinfo...     │
└──────────┘     └─────────────────┘
```

| 概念 | 说明 |
|------|------|
| **vptr** | 编译器在**有虚函数的对象开头**插入的隐藏指针 |
| **vtable** | 每个有虚函数的类一张**静态**函数指针表 |
| **调用** | `p->foo()` → 读 vptr → 查 vtable[index] → 间接跳转 |
| **构造顺序** | 构造 Base 时 vptr 指向 Base vtable；Derived 构造完成后才指向 Derived vtable |

### 高频问答

**Q: 为什么 virtual 有开销？**

- 每次调用：2 次间接寻址 + 间接分支 → ICache miss / 分支预测失败
- 每对象 +8 bytes vptr
- 阻碍内联（除非 `final`、devirtualize、LTO）

**Q: 为什么 Destructor 要 virtual？**

```cpp
Base* p = new Derived();
delete p;  // 若 ~Base 非 virtual → 只调 Base 析构 → Derived 成员泄漏（UB）
```

**只要类可能被基类指针 delete，析构就必须 virtual。**

**Q: override / final？**

```cpp
void foo() override;   // 显式标记覆盖，拼写错误编译报错
void bar() final;      // 禁止进一步 override
class Sealed final {}; // 禁止继承
```

**Q: 纯虚函数 / 抽象类？**

```cpp
class Shape {
public:
    virtual double area() const = 0;  // 纯虚
    virtual ~Shape() = default;
};
// 不能实例化 Shape；Derived 必须实现 area()
```

## 四种 cast（高频 FAQ）

| cast | 用途 | 安全性 |
|------|------|--------|
| **static_cast** | 相关类型转换（数值、父子类向上转型） | 编译期检查，无运行时开销 |
| **dynamic_cast** | 多态下行转型 | 运行时检查，失败返回 `nullptr`（指针）或抛异常（引用） |
| **const_cast** | 添加/移除 const/volatile | 仅改 cv 限定；**不能改类型** |
| **reinterpret_cast** | 位级重新解释（指针↔整数、无关类型） | 危险，平台相关 |

```cpp
// static_cast：编译器已知继承关系
Derived* d = static_cast<Derived*>(basePtr);  // 无运行时检查

// dynamic_cast：需要虚函数（RTTI）
Derived* d2 = dynamic_cast<Derived*>(basePtr);  // 失败 → nullptr

// const_cast
void legacy(char* p);
void use(const char* s) {
    legacy(const_cast<char*>(s));  // 仅当 legacy 不写才安全
}

// reinterpret_cast：底层/硬件
uintptr_t addr = reinterpret_cast<uintptr_t>(ptr);
```

**面试原则：**

- 优先 `static_cast`（明确、可搜索）
- 多态下行用 `dynamic_cast`
- 避免 `reinterpret_cast` 除非硬件/序列化
- `const_cast` 极少用，通常是接口遗留问题

## CRTP 替代虚函数（加分）

热路径避免 vcall：

```cpp
template <typename Derived>
class Base {
public:
    void interface() {
        static_cast<Derived*>(this)->impl();  // 编译期绑定，可内联
    }
};
class Fast : public Base<Fast> {
public:
    void impl() { /* ... */ }
};
```

AWS Infra 热路径（包处理、每包回调）常考「何时用虚函数 vs 模板 vs function pointer」。
