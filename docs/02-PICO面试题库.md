## 一、手撕 / 编程题（高频）

字节整体偏 **LeetCode Medium+**，PICO 视觉岗还会加 **CV 专属手撕**。  
代码参考：[`interview_handwrite/`](../interview_handwrite/)（与下文高度重合）

### 1. LeetCode 类（笔试 + 面必考）

| 类型 | 代表题目 |
|------|----------|
| 双指针 / 滑动窗口 | 最长不重复子串、最小覆盖子串、滑动窗口最大值 |
| 动态规划 | 最大子数组和、背包、编辑距离、路径 DP |
| 链表 | 删除倒数第 K 个节点、反转链表、环检测 |
| 二叉树 | 层序遍历、最近公共祖先、路径和 |
| 图 / 搜索 | BFS/DFS、课程表（拓扑排序）、岛屿数量 |
| 字符串 | 最长回文串/子串、KMP 思想 |
| 设计题 | LRU Cache、合并 K 个有序链表 |

**字节特色：** 飞书文档手写、无自动补全；**先讲思路再写代码**。

### 2. CV 专属手撕（PICO 视觉岗极高频）

| 题目 | 考察点 |
|------|--------|
| 2D 卷积 | padding/stride、多通道累加、im2col |
| NMS | IoU 计算、贪心抑制、per-class NMS |
| 双线性插值 | 4 邻域权重、resize / grid_sample |
| Softmax / ReLU | 数值稳定（减 max） |
| IoU / GIoU | 检测后处理 |
| 矩阵乘法 | 三重循环或分块思想 |
| 时间戳对齐插值 | IMU-相机融合岗常考 |
| ICP / PnP | 追踪/SLAM 岗 |

详见 [04-手撕代码指南.md](./04-手撕代码指南.md) 与 [03-算法数学公式.md](./03-算法数学公式.md)。

### 3. C++ 基础（5 年经验岗常考）

- 智能指针、虚函数/多态、move 语义  
- `vector` / `map` 使用与复杂度  
- 多线程基础、`std::mutex`  
- Eigen 矩阵运算基本接口

## 二、项目深挖（几乎每轮必问）

面试官会把你简历上的项目问到 **无法糊弄** 的深度。

| 问题类型 | 典型问题 |
|----------|----------|
| 动机 | 为什么选这个网络？baseline 是什么？ |
| 数据 | 数据怎么采集/标注？多少张？bad case 有哪些？ |
| 训练 | loss 怎么设计？难例怎么挖？过拟合怎么处理？ |
| 指标 | 客观指标（PSNR/mAP）和主观指标怎么对齐？ |
| 部署 | 量化后精度掉多少？怎么救？端到端延迟多少 ms？ |
| 权衡 | 为什么放端侧不放云端？内存峰值多少？ |
| 协作 | 和相机/ISP/硬件团队怎么对齐规格？ |

**PICO 特别爱问：**

> 「从相机采集到用户看到结果，整条 pipeline 延迟怎么分配？」

**答题建议：** 用 STAR 法则，准备 latency budget 表格（每模块 ms + 占比）。

## 六、字节视觉面经真题示例

1. Transformer：self-attention、positional encoding、cross-attention；预测时如何 autoregressive？  
2. VAE：隐空间为何接近 N(0,1)？KL 散度公式？  
3. VAE vs GAN：训练稳定性、生成清晰度 trade-off  
4. CLIP / BLIP / 多模态  
5. 目标检测：Faster R-CNN、YOLO、DETR（为何 DETR 可不用 NMS）  
6. 1×1 卷积作用  
7. ViT / Swin 结构  
8. 算法题：最长不重复子串、删除倒数第 K 个节点、二维矩阵每行选一个使和最大（DFS/分配类）
