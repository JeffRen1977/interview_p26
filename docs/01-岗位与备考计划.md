# 01 - 岗位解读与备考计划

## 岗位画像

**团队：** PICO Imaging Systems & Human Engineering

**核心使命：** capture-to-display 全链路影像智能，连接相机系统、显示技术与人眼视觉感知。

```
Camera / ISP / 光学 → 计算摄影 / 增强复原 → CV 感知 / 场景理解 → 大模型语义 → 端云协同 → Display / 人眼感知
```

| 维度 | 要求 |
|------|------|
| 业务 | 图像增强、复原、场景理解、检测分类、视觉推理 |
| 工程 | 端侧实时、低功耗、低内存；量化/压缩/硬件加速 |
| 架构 | 端侧 + 云端大模型协同；延迟/隐私/鲁棒性权衡 |
| 协作 | 相机、显示、平台、产品、供应商联合设计与验证 |
| 经验 | 5+ 年相关经验；消费电子产品落地加分 |

## 需要准备的材料

### 1. 简历（1.5–2 页）

每条经历用 **「问题 → 方法 → 指标 → 约束」** 写，必须出现：

- 端侧 AI / 模型部署 / 量化 / 蒸馏 / 剪枝
- TensorRT / NCNN / ONNX Runtime（实际用过的）
- 计算摄影 / ISP / 检测 / 分割 / 深度 / 场景理解
- 具体数字：延迟 ms、功耗 mW、内存 MB、PSNR/mAP 等
- XR / 移动端 / 消费电子（如有）

**Bullet 模板：**

> Led [算法] for [产品/场景], achieving [指标] on [芯片/平台]. Collaborated with [相机/ISP/显示] teams. [量化/剪枝] reduced latency from Xms to Yms with <Z% accuracy drop.

### 2. 王牌项目 Dossier（8–10 页，自用复习）

1. 业务背景与用户痛点  
2. 系统架构图（Camera → ISP → CV → [Cloud?] → Display，标注 latency budget）  
3. 算法设计与选型理由（RAW vs RGB 域等）  
4. 数据与评测（客观 PSNR/SSIM/LPIPS + 主观 MOS）  
5. 端侧部署（PTQ/QAT、框架、优化前后对比）  
6. 端云权衡（隐私/延迟/带宽）  
7. 跨团队协作与联调踩坑  
8. 量产结果与影响  

### 3. 作品集（GitHub + Demo）

推荐项目：**Capture-to-Display 画质增强 Pipeline**

- 输入：RAW 或低质量 RGB  
- 处理：去噪 → 增强 →（可选）语义分割辅助 → tone mapping  
- 部署：PyTorch → ONNX → TensorRT/NCNN  
- 产出：对比图、benchmark 表、30s demo 视频  

### 4. 行为面 STAR 故事（3 个）

- 精度 vs 延迟的技术取舍  
- 与硬件/ISP 规格冲突的协作  
- 从 research 到量产的完整经历

## 技术知识矩阵

### Tier 1 — 必考

| 领域 | 要点 |
|------|------|
| 计算摄影 | 去噪、HDR、超分、去模糊、低光增强 |
| 端侧部署 | INT8 PTQ/QAT、TensorRT/NCNN、算子融合、内存优化 |
| CV 基础 | 检测（YOLO）、分割（DeepLab）、分类、深度估计 |
| 系统权衡 | 端 vs 云：延迟、隐私、带宽、离线能力 |

### Tier 2 — 强加分

| 领域 | 要点 |
|------|------|
| ISP | RAW pipeline、demosaic、NR、3A、AI-ISP |
| 光学/相机 | 标定、畸变校正、rolling shutter、多相机同步 |
| 人眼视觉 | 色彩空间 sRGB/P3、gamma、MOS 评测 |
| 大模型视觉 | CLIP、VLM、端云分流 |

### Tier 3 — 差异化

| 领域 | 要点 |
|------|------|
| VR/MR | See-through 透视、VST 画质、虚实对齐 |
| Display | 畸变校正、注视点渲染 |
| 3D | NeRF/3DGS、SLAM 基本概念 |
