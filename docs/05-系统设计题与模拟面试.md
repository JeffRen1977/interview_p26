# 05 - 系统设计题与模拟面试

## 一、Capture-to-Display 系统设计（MR 透视标准答案）

**题目：** 设计 PICO MR 设备的 see-through 透视系统，从相机采集到用户看到增强后的实景，如何架构？

### 1. 需求拆解

| 维度 | 典型目标 |
|------|----------|
| 分辨率 | 单眼 1920×1080 或更高 |
| 帧率 | 72/90 Hz（与显示刷新同步） |
| 端到端延迟 | < 20–30 ms（MTP 敏感，超过易晕） |
| FOV | 与显示 FOV 匹配，减少黑边 |
| 画质 | 低畸变、色彩准确、低噪声、运动清晰 |

### 2. 硬件约束

- 前置 RGB 相机 × 2+（立体/广角）  
- 算力：骁龙 XR 芯片，NPU + GPU + DSP  
- 功耗/热：持续 MR 模式不能明显降频  
- 显示：LCD/OLED，需畸变与色差校正  

### 3. Pipeline 架构

```
[Camera Sync] → [ISP/RAW处理] → [畸变校正] → [深度/对齐] → [增强] → [虚实融合] → [显示校正] → [Display]
     1ms            3ms            2ms          5ms         4ms        2ms           3ms        —
```

### 4. 延迟预算表示例（总计 ~20ms）

| 模块 | 预算 | 说明 |
|------|------|------|
| 采集 + sync | 1–2 ms | 多相机时间戳对齐 |
| ISP / debayer | 2–4 ms | RAW 或 YUV，部分可走硬件 ISP |
| 畸变校正 | 1–2 ms | 查表 remap + 双线性插值 |
| 深度估计 | 4–6 ms | 双目匹配或轻量单目网络 |
| 虚实对齐 | 2–3 ms | 深度图 reproject 到显示坐标 |
| 画质增强 | 3–5 ms | 去噪、色调映射、锐化（可 INT8） |
| 显示 pipeline | 2–3 ms | 畸变/色差校正、合成 |
| **合计** | **~18–22 ms** | 需留 buffer 应对抖动 |

### 5. 端云分工

| 任务 | 端侧 | 云端 | 理由 |
|------|------|------|------|
| 实时透视渲染 | ✅ | ❌ | 延迟、离线 |
| 语义场景理解 | 轻量分类 | VLM 详细描述 | 端侧做 trigger，云端异步 |
| 地图/物体识别 | 缓存 + 端侧检索 | 大库更新 | 隐私 + 带宽 |
| 模型 OTA | — | ✅ | 非实时 |

### 6. 评测方案

**客观：** PSNR/SSIM/LPIPS（增强前后）、深度误差、重投影误差、延迟分布（p50/p99）

**主观：** MOS 打分（清晰度、色彩、舒适度、晕动症）

**场景：** 室内/室外、低光、快速转头、近距/远距

### 7. 失败 Fallback

| 场景 | 策略 |
|------|------|
| 低光 | 多帧融合 + 传统 NR fallback |
| 运动模糊 | 降增强强度，优先保延迟 |
| 深度失效 | 固定平面假设或上一帧深度 |
| 算力不足 | 降分辨率 / 跳帧增强 / 关云端 |
| 追踪丢失 | 保持最后一帧 + 渐变提示 |

## 二、其他高频系统设计题

### Q1：如何在 50ms 内完成手部追踪全流程？

```
采集(2) → 预处理(3) → 检测/ROI(5) → 关键点网络(15) → 后处理+滤波(5) → 输出(2)
= 32ms，留 18ms 给系统调度
```

- 网络：MobileNet / 专用轻量 backbone + INT8  
- ROI：上一帧热启动，减少搜索范围  
- 时序：Kalman/One-Euro 滤波平滑  

### Q2：端侧模型 INT8 量化后 PSNR 掉 2dB，怎么救？

1. 敏感层保持 FP16（首层、末层、小 channel 层）  
2. QAT 替代 PTQ  
3. 蒸馏：FP32 teacher → INT8 student  
4. 扩大校准集，覆盖低光/高动态场景  
5. per-channel 量化  

### Q3：如何设计端侧场景检测的数据集和评测？

- 数据：覆盖居家/办公/户外，光照/遮挡/运动模糊  
- 标注：场景类别 + 难例标签  
- 指标：mAP + 端侧延迟 + 功耗 + 长尾类 recall  
- 上线：A/B + 用户反馈闭环

## 三、模拟面试流程（90 min）

### 环节 1：项目深挖（25 min）

准备 15–20 分钟口述，面试官可能追问：

1. 你的贡献是什么？  
2. 为什么不用更大的模型？  
3. 量化前后数字对比？  
4. 最大的技术坑是什么？  

### 环节 2：基础原理（15 min）

随机 3–5 题，例如：

- BN 训练和推理的区别？  
- YOLO v5/v8 为什么快？  
- RAW 域做网络有什么好处？  

### 环节 3：手撕代码（25 min）

任选 2 道：

1. NMS  
2. 双线性插值  
3. 2D 卷积（单通道）  
4. Softmax  

### 环节 4：系统设计（20 min）

任选 1 道：

1. MR see-through pipeline（本文第一节）  
2. 端云协同的视觉理解架构  
3. 低光视频增强系统  

### 环节 5：反问（5 min）

- 团队当前最关心的技术挑战？  
- 算法到量产的典型周期？  
- 端侧主要用什么推理框架？
