## 一、CNN

### Q1：1×1 卷积的作用是什么？

**核心：** 1×1 卷积只在通道维度做线性组合，不改变特征图的空间尺寸（H×W 不变）。

**主要作用：**

| 作用 | 说明 |
|------|------|
| **跨通道信息融合** | 对每个空间位置，把所有输入通道加权求和，得到新的通道表示 |
| **升降维（瓶颈）** | Inception、ResNet bottleneck 中先 1×1 降维再 3×3 卷积，减少计算量 |
| **增加非线性** | 1×1 conv + ReLU 在不改变分辨率的情况下增加网络深度和表达能力 |
| **通道对齐** | 检测/分割 head 中把 backbone 特征通道数映射到任务所需维度 |

**计算量对比（面试加分）：**

- 3×3 conv on $C \to C$: $9C^2HW$
- 先 1×1 降到 $C/4$，再 3×3，再 1×1 升回 $C$: 约 $\frac{1}{4}C^2HW + \frac{9}{16}C^2HW + \frac{1}{4}C^2HW$，显著更省

**一句话总结：** 1×1 卷积 = 逐像素的全连接层，负责通道混合与维度变换，不负责扩大感受野。

### Q2：感受野（Receptive Field）怎么计算？

**定义：** 输出特征图上某一点「能看到」的输入图像区域大小。

**单层卷积（kernel=k, stride=s, padding=p）：**

从第 $l$ 层到第 $l-1$ 层的递推公式：


$$
RF_{l-1} = (RF_l - 1) \times s_l + k_l
$$



$$
j_{l-1} = j_l \times s_l
$$


（$j$ 为相邻感受野中心间距，初始 $RF_L=1, j_L=1$，从最后一层往前推）

**常用结论（stride=1, padding 保持尺寸时）：**

堆叠 $L$ 层 kernel 为 $k$ 的卷积：


$$
RF \approx 1 + L(k-1)
$$


**例题：** 3 层 3×3 卷积，stride=1 → $RF = 1 + 3\times2 = 7$，等价于一个 7×7 卷积的感受野。

**有 stride 或 pooling 时：** 必须用递推公式；pooling(k, s) 与 conv 形式相同。

**面试要点：**
- 感受野大 → 语义信息多，但空间细节少
- FPN、空洞卷积、大 kernel 都是为了在不降分辨率的情况下扩大感受野
- 实际「有效感受野」往往小于理论值（中心权重更大）

### Q3：Batch Normalization（BN）原理？

**动机：** 解决内部协变量偏移（Internal Covariate Shift）——每层输入分布随训练变化，导致收敛慢、对学习率敏感。

**训练时（mini-batch）：**

对 batch 中每个通道 $c$，在 $N \times H \times W$ 个元素上统计：


$$
\mu_c = \frac{1}{NHW}\sum x_i,\quad
\sigma_c^2 = \frac{1}{NHW}\sum (x_i - \mu_c)^2
$$


归一化 + 可学习仿射变换：


$$
\hat{x} = \frac{x - \mu_c}{\sqrt{\sigma_c^2 + \epsilon}},\quad
y = \gamma_c \hat{x} + \beta_c
$$


$\gamma, \beta$ 可学习，使网络在需要时可以恢复恒等映射。

**推理时：** 用训练阶段累积的 **running mean / running variance**，不用当前 batch 统计量。

**效果：**
- 允许更大学习率，加速收敛
- 轻微正则化效果
- 对初始化不那么敏感

**变种（端侧/小 batch 常问）：**
- **Layer Norm**：在通道维归一化，用于 Transformer
- **Group Norm**：不依赖 batch size，小 batch / 检测常用
- **Sync BN**：多卡同步 batch 统计量

**面试陷阱：** BN 在 batch=1 或极小 batch 时统计不准；端侧推理等价于用固定 $\mu,\sigma$ 的仿射变换。

## 二、检测（Detection）

### Q4：YOLO 为什么快？

**一句话：** 把检测变成**单次前向**的回归问题，端到端，无需复杂的两阶段 pipeline。

| 对比维度 | YOLO | Faster R-CNN |
|----------|------|--------------|
| 阶段 | 1-stage | 2-stage |
| 候选框 | 固定 grid/anchor 直接预测 | RPN 生成 ~2000 proposals |
| 推理次数 | 1 次 backbone 前向 | backbone + RPN + 逐 ROI 池化 + head |
| NMS 输入 | 较少（每 grid 少量框） | 较多 proposals |

**YOLO 快的具体原因：**

1. **单次检测：** 图像 → CNN → 直接输出 $(x,y,w,h,\text{conf},\text{cls})$，无 ROI 裁剪循环
2. **全卷积：** 除 NMS 外全程 GPU 并行
3. **轻量 backbone：** YOLOv5/v8 用 CSPDarknet、C2f 等效率结构
4. **Anchor-free 趋势（v8）：** 减少 anchor 设计复杂度
5. **工程优化：** TensorRT、FP16/INT8、算子融合

**代价：** 小目标、密集目标 historically 弱于两阶段；近年 YOLO 已大幅缩小差距。

### Q5：Faster R-CNN 两阶段流程？

```
输入图像
    ↓
Backbone (ResNet等) → 特征图
    ↓
RPN (Region Proposal Network) → ~2000 候选框 + objectness
    ↓
Proposal 筛选 (NMS, top-K)
    ↓
ROI Pooling / ROI Align → 固定大小特征 (7×7)
    ↓
检测 Head → 分类 + 边框回归
    ↓
最终 NMS → 输出
```

**Stage 1 — RPN：**
- 在特征图上滑动，每个位置生成多个 anchor
- 预测：objectness（是否物体）+ anchor 偏移量
- 输出候选区域（proposals）

**Stage 2 — 检测头：**
- 对每个 proposal 用 ROI Align 提取固定维特征（双线性插值，比 ROI Pool 更精确）
- 分类（什么类别）+ 边框精修（二次回归）

**为什么准：** 两阶段分工——RPN 粗定位，检测头在裁剪后的区域做精细分类和回归；ROI Align 保留空间精度。

**慢在哪：** 要对大量 proposals 逐个做 ROI 操作，无法像 YOLO 那样一次输出。

---

### Q6：NMS 原理？

详见 [03-算法数学公式.md](./03-算法数学公式.md#2-nms非极大值抑制) 与 [interview_handwrite/nms.py](../interview_handwrite/nms.py)。

**口述版：**

1. 计算框之间 **IoU** = 交集 / 并集
2. 按置信度**从高到低**排序
3. 取最高分框保留，删除与它 IoU > 阈值（如 0.5）的所有框
4. 重复直到没有剩余框

**要点：**
- 多类别检测要做 **per-class NMS**
- 只在**推理**时用，训练不用（不可微）
- Soft-NMS：重叠框降分而非直接删除

## 三、分割（Segmentation）

### Q7：FCN / DeepLab / SegFormer 区别？

| | FCN | DeepLab 系列 | SegFormer |
|---|-----|--------------|-----------|
| **年代** | 2015 | 2015–2018+ | 2021 |
| **核心思想** | 全卷积 + 上采样 | 空洞卷积 + ASPP | Transformer encoder + 轻量 MLP decoder |
| **Backbone** | VGG 等 CNN | ResNet / Xception | MiT (Mix Transformer) |
| **多尺度** | skip connection 融合 | Atrous Spatial Pyramid Pooling | 多阶段特征融合 |
| **上采样** | 反卷积 / bilinear | 双线性 + 空洞卷积 | MLP 上采样 |
| **优点** | 开创端到端语义分割 | 多尺度上下文、精度高 | 无 positional encoding 插值问题、结构简洁 |
| **缺点** | 边界粗糙 | 计算量大、超参多 | 大模型计算量高 |
| **典型应用** | 学术 baseline | 移动端/服务器语义分割 | 高精度场景分割 |

**FCN 关键点：** 把全连接层换成卷积，任意尺寸输入；用 skip link 融合浅层细节和深层语义。

**DeepLab 关键点：**
- **空洞卷积（Atrous Conv）：** 扩大感受野不降低分辨率
- **ASPP：** 并行多个不同 dilation rate 的空洞卷积，捕获多尺度上下文
- **DeepLabv3+：** encoder-decoder + skip connection

**SegFormer 关键点：**
- 纯 Transformer encoder，分层输出特征
- Decoder 只用 MLP 融合多级特征，不用复杂解码器
- 无 heavy decoder，效率与精度平衡好

**面试一句话：**
- FCN：卷积化 + 上采样开山之作
- DeepLab：空洞卷积 + ASPP 多尺度
- SegFormer：Transformer + 轻量 MLP decoder 的现代方案

## 四、Transformer

### Q8：Self-Attention 公式？

**输入：** $X \in \mathbb{R}^{N \times d}$（$N$ 个 token，维度 $d$）

**线性投影：**


$$
Q = XW_Q,\quad K = XW_K,\quad V = XW_V
$$


**Scaled Dot-Product Attention：**


$$
\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
$$


- $QK^T$：每对 token 的相似度，形状 $N \times N$
- $\sqrt{d_k}$：缩放因子，防止点积过大导致 softmax 梯度消失
- softmax 后每行和为 1，是对 $V$ 的加权求和

**复杂度：** $O(N^2 d)$，序列长时成为瓶颈（Swin、线性 attention 等优化方向）。

---

### Q9：多头注意力（Multi-Head Attention）？

把 $d$ 维拆成 $h$ 个头，每头维度 $d_k = d/h$：


$$
\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)
$$



$$
\text{MultiHead} = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O
$$


**为什么多头：**
- 不同 head 学习不同的注意力模式（局部纹理 vs 全局结构）
- 类似 CNN 多通道，增强表达能力

**面试常问：** Self-attention vs Cross-attention
- **Self-attention：** $Q,K,V$ 来自同一序列
- **Cross-attention：** $Q$ 来自 decoder，$K,V$ 来自 encoder（如 DETR、VLM）

---

### Q10：ViT / Swin 结构？

#### ViT (Vision Transformer)

```
图像 → 切成 P×P patches → 展平 + Linear embedding
    → 加 [CLS] token + 位置编码
    → L 层 Transformer Encoder (MSA + MLP)
    → [CLS] 或 patch tokens → 分类 / 下游任务
```

**要点：**
- 把图像当序列，每个 patch 是一个 token
- 需要大数据预训练（JFT-300M）才能超过 CNN
- 全局 attention，复杂度 $O(N^2)$，$N = (H/P)(W/P)$

#### Swin Transformer

```
图像 → patch embedding
    → Stage 1: Swin Block × N  (Window MSA, 局部窗口)
    → Patch Merging (下采样 2×)
    → Stage 2, 3, 4 ... (窗口逐渐变大，感受野增大)
    → 多尺度特征 → 检测/分割 head
```

**Swin 关键创新：**
- **Window-based MSA：** 只在 $M \times M$ 窗口内做 attention，$O(N)$ 复杂度
- **Shifted Window：** 相邻层窗口错开，实现跨窗口信息交互
- **层级结构：** 类似 CNN 金字塔，适合密集预测（检测、分割）

| | ViT | Swin |
|---|-----|------|
| Attention 范围 | 全局 | 局部窗口 + shift |
| 复杂度 | $O(N^2)$ | $O(N)$ |
| 适合任务 | 分类 | 分类 + 检测 + 分割 |
| 归纳偏置 | 弱，需大数据 | 有局部性偏置 |

## 五、生成模型

### Q11：VAE vs GAN？

| | VAE | GAN |
|---|-----|-----|
| **框架** | 概率生成模型，显式似然 | 博弈：Generator vs Discriminator |
| **训练目标** | ELBO（重建 + KL 正则） | min-max 对抗损失 |
| **隐空间** | 连续、有结构，可插值 | 无显式推断，隐空间通过噪声采样 |
| **生成质量** | 偏模糊（MSE 重建损失） | 通常更锐利 |
| **训练稳定性** | 较稳定 | 不稳定（模式崩塌、D/G 不平衡） |
| **推断** | 有 encoder，可做表征学习 | 一般只有 generator |
| **应用** | 压缩、表征、可控生成 | 图像生成、风格迁移、超分 |

**面试总结：**
- VAE：要**可解释的隐空间**和稳定训练选 VAE
- GAN：要**视觉质量**选 GAN（或扩散模型）
- 现代趋势：Diffusion 在质量与稳定性上超越两者

### Q12：KL 散度是什么？VAE 里为什么用？

**定义（离散）：**


$$
D_{KL}(P \| Q) = \sum_i P(i) \log \frac{P(i)}{Q(i)}
$$


**连续（高斯之间常用）：**


$$
D_{KL}(\mathcal{N}(\mu,\sigma^2) \| \mathcal{N}(0,1)) = \frac{1}{2}(\mu^2 + \sigma^2 - \log\sigma^2 - 1)
$$


**含义：** 衡量分布 $P$ 相对 $Q$ 的「额外信息量」；非负，为 0 当且仅当两分布相同。

**VAE 中的作用：**


$$
\mathcal{L} = \underbrace{\|x - \hat{x}\|^2}_{\text{重建损失}} + \underbrace{D_{KL}(q(z|x) \| p(z))}_{\text{正则项}},\quad p(z)=\mathcal{N}(0,I)
$$


- 迫使 encoder 输出的隐分布 $q(z|x)$ 接近标准正态 $\mathcal{N}(0,I)$
- 这样**生成时**可以从 $\mathcal{N}(0,I)$ 采样 $z$，再 decoder 出图像
- 没有 KL：隐空间不规则，随机采样无法生成合理图像

---

### Q13：重参数化技巧（Reparameterization Trick）？

**问题：** 从 $q(z|x) = \mathcal{N}(\mu, \sigma^2)$ 采样 $z$ 不可微，无法反向传播。

**技巧：**


$$
z = \mu + \sigma \cdot \epsilon,\quad \epsilon \sim \mathcal{N}(0, I)
$$


- 随机性转移到 $\epsilon$（与参数无关）
- 梯度可以通过 $\mu, \sigma$ 回传

**面试一句话：** 把随机采样改写成确定性函数 + 独立噪声，使 VAE 的 ELBO 可端到端训练。

## 六、多模态

### Q14：CLIP 对比学习？

**CLIP (Contrastive Language-Image Pre-training)** — OpenAI，2021

**架构：**

```
图像 → Image Encoder (ViT/ResNet) → 图像特征 I
文本 → Text Encoder (Transformer)  → 文本特征 T
```

**训练（对比损失）：**

一个 batch 有 $N$ 对 (image, text)，计算 $N \times N$ 相似度矩阵：


$$
\mathcal{L} = -\frac{1}{N}\sum_i \log \frac{\exp(I_i \cdot T_i / \tau)}{\sum_j \exp(I_i \cdot T_j / \tau)}
$$


- 对角线是正样本（匹配的图文对）
- 同 batch 内其他配对是负样本
- $\tau$：温度系数，控制分布锐度

**效果：**
- 学到对齐的图文嵌入空间
- **Zero-shot 分类：** 把类别名变成 prompt（"a photo of a dog"），与图像特征比相似度
- 成为后续 VLM 的视觉编码器基础

**面试要点：**
- 大规模图文对（4亿）是关键
- 对比学习 vs 生成式：CLIP 不生成图像，只对齐表征
- 局限：细粒度理解、计数、空间关系较弱

### Q15：BLIP / VLM 基本概念？

#### BLIP (Bootstrapping Language-Image Pre-training)

解决 CLIP 的**噪声图文对**问题，三阶段：

1. **Captioner：** 生成图像描述
2. **Filter：** 过滤噪声 caption
3. **多任务预训练：** 对比学习 + 图像-文本匹配 (ITM) + 语言建模 (LM)

比纯 CLIP 在检索和 caption 上更强。

#### VLM (Vision-Language Model) 通用架构

```
图像 → Vision Encoder (ViT/CLIP) → 视觉 token
文本 → Tokenizer → Text Embedding
         ↓
    跨模态融合 (Cross-Attention / Q-Former / projector)
         ↓
    LLM (LLaMA/Qwen 等) → 文本输出
```

**代表模型：**

| 模型 | 特点 |
|------|------|
| **LLaVA** | 简单 linear projector 接 LLM，指令微调 |
| **BLIP-2** | Q-Former 桥接冻结的视觉编码器和 LLM |
| **GPT-4V / Qwen-VL** | 大规模多模态预训练 + 对话能力 |

**PICO 相关考点（端云协同）：**
- 端侧：轻量视觉 encoder + 小分类器做 trigger
- 云端：VLM 做复杂语义理解（场景描述、物体推理）
- 隐私敏感内容不上云，仅上传特征或结构化结果

**面试一句话：**
- CLIP：图文对比对齐，zero-shot 基础
- BLIP：去噪 + 多任务，更强的图文理解
- VLM：视觉 encoder + LLM，实现看图说话、视觉问答
