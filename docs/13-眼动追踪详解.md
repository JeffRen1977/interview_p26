# 13 - XR 眼动追踪（Eye Tracking）详解

XR 眼镜/头显中的 **眼球跟踪（Eye Tracking）** 技术：从近眼红外成像到 Gaze 估计、注视点渲染与眼动交互。

> **GitHub 阅读：** 公式使用 `$...$` / `$$...$$` 格式。  
> 相关：[09-追踪交互题详解.md](./09-追踪交互题详解.md)、[12-3D人脸表情追踪详解.md](./12-3D人脸表情追踪详解.md)

---

## 一、核心目标与输出

从近眼相机采集的 **NIR 红外图像** 中，实时估计：

| 输出 | 说明 | 典型用途 |
|------|------|----------|
| **Gaze 方向** | 3D 视线向量或显示平面落点 $(x,y)$ | 注视点渲染、UI 选中 |
| **瞳孔中心** | 图像/3D 坐标 | 底层特征 |
| **Glint（角膜反射）** | 红外 LED 反光点 | 几何解算、标定 |
| **眼睑 / 眨眼** | 开合度 | Avatar、疲劳检测 |
| **IPD（瞳距）** | 两眼瞳孔距离 | 光学对齐、舒适度 |
| **Vergence** | 双眼汇聚角 | 立体深度、调焦 |

**延迟目标：** 端到端 often **< 10–20 ms**，配合 Foveated Rendering 需 **Gaze Prediction**。

---

## 二、硬件方案

XR 眼动几乎均采用 **近眼红外（NIR）相机 + NIR LED**，而非可见光。

| 方案 | 说明 | 代表 |
|------|------|------|
| **环视 IR 相机 + LED** | 每眼 1 相机，旁侧打光，拍瞳孔+glint | Quest Pro、PICO 高端、Vive Pro Eye、Varjo |
| **多 LED / 结构光** | 多个 glint，几何约束更强 | 科研、高端方案 |
| **事件相机** | 高时间分辨率 | 前沿研究 |
| **EOG 眼电图** | 电极测电位 | 少见，佩戴不适 |

**为何用红外：**
- 瞳孔在 NIR 下与虹膜对比度高（亮瞳/暗瞳）
- 不干扰用户观看屏幕内容
- 不受显示画面颜色影响

---

## 三、算法 Pipeline

```
近眼 NIR 图像（每眼，60–120 Hz）
    ↓
[1] 预处理]     去噪、阈值、红外均衡
    ↓
[2] 瞳孔检测]   椭圆/轮廓/CNN → 中心 (cx, cy)
    ↓
[3] Glint 检测] 高亮斑点 → 角膜反射点
    ↓
[4] 特征]       瞳孔-glint 向量、边缘、深度特征
    ↓
[5] Gaze 估计]  3D 视线或屏幕落点
    ↓
[6] 双眼融合]   左右眼合并
    ↓
[7] 滤波/预测]  Kalman、Gaze Prediction
    ↓
[8] 标定]       用户 9/13 点个性化
    ↓
输出 gaze(x,y), confidence, IPD
```

---

## 四、核心算法

### 4.1 传统几何法（Model-Based）

基于眼球光学模型：球体 + 角膜 + 瞳孔。

**特征：**
- 瞳孔中心 $\vec{p}_{pupil}$
- 1–4 个 glint $\vec{p}_{glint}$
- 向量 $\vec{v} = \vec{p}_{pupil} - \vec{p}_{glint}$

**步骤：**
1. 角膜球面模型估计光轴  
2. 映射到**视觉轴（Visual Axis）**（与瞳孔光学轴差约 5–15°，个体化）  
3. 结合头显几何变换到显示坐标系  

**优点：** 可解释、算力低  
**缺点：** 依赖 glint、个体解剖差异、滑移（slippage）

---

### 4.2 瞳孔检测（底层共性）

| 方法 | 说明 |
|------|------|
| 阈值 + 轮廓 | Otsu + 最大连通域 |
| 椭圆拟合 RANSAC | 中心、长短轴 |
| Starburst / Swirski | 射线向外找边界 |
| CNN 分割 | U-Net 输出 pupil mask |
| 直接回归 | 输出 $(c_x, c_y, r)$ |

**难点：** 睫毛、半闭眼、眼镜反光、虹膜对比度差异。

---

### 4.3 回归映射法（ML Feature-Based）

$$
\mathbf{g} = f_\theta(\mathbf{x}_{pupil}, \mathbf{x}_{glint}, \ldots)
$$

或屏幕落点：

$$
\mathbf{g}_{screen} = \mathbf{A} \cdot \mathbf{f} + \mathbf{b}
$$

**用户标定（必做）：** 看 9/13 个校准点，拟合个性化 $\mathbf{A}, \mathbf{b}$。

**代表：** Tobii、Pupil Labs、多数商用 SDK。

---

### 4.4 端到端深度学习（Appearance-Based）

```
NIR 图像 → CNN / ViT → (pitch, yaw) 或 (x, y)
```

| 优点 | 缺点 |
|------|------|
| 少手工特征 | 需大数据 |
| 遮挡更鲁棒 | 跨设备泛化难 |

**增强：** 时序 LSTM/TCN、双眼联合网络、few-shot 个性化微调。

---

### 4.5 3D 眼球模型优化（Optimization-Based）

$$
\min_{\mathbf{g}, \mathbf{s}} \sum_i \| \pi(\mathbf{g}, \mathbf{s}) - \mathbf{u}_i \|^2
$$

高精度科研/医疗；XR 实时多与 CNN 检测 hybrid。

---

### 4.6 滑移补偿（Slippage Compensation）

眼动传感器相对头显微动（佩戴松紧）：

| 方法 | 说明 |
|------|------|
| Glint 稳定坐标系 | 以 glint 为参考 |
| 头姿补偿 | IMU/SLAM 辅助 |
| 在线 mini 标定 | 检测漂移后重校 |

---

### 4.7 时序滤波与 Gaze Prediction

| 现象 | 处理 |
|------|------|
| 生理性微跳 | Kalman / One-Euro |
| 扫视 Saccade | 高速段检测，降权或外推 |
| 渲染延迟 | **预测 20–50ms 后注视点** |

Foveated Rendering 强烈依赖 gaze prediction，否则外围降分辨率区会「拖影」。

## 五、XR 主要应用

| 应用 | 说明 |
|------|------|
| **注视点渲染（Foveated Rendering）** | 注视区全分辨率，外围降采样，省 30–50% GPU |
| **可变焦显示** | 结合 vergence 调焦，减眩晕（前沿） |
| **眼动交互** | Dwell 凝视点击、菜单 |
| **IPD 自动测量** | 个体光学间距 |
| **Avatar 眼神** | 社交 VR |
| **UX / 疲劳分析** | 注意力热图 |

## 六、与面部追踪的关系

| 模块 | 传感器 | 输出 | 精度 |
|------|--------|------|------|
| **面部追踪** | RGB 内视 | blendshape、`eyeLook*` 粗 gaze | 表情级 |
| **眼动追踪** | 近眼 NIR | 高精度 gaze、IPD | ~0.5–1.5° |

## 七、端侧工程挑战

| 难点 | 对策 |
|------|------|
| 眼镜/睫毛遮挡 | 数据增强、降权、IR 角度优化 |
| 个体差异 | 9/13 点标定 + 在线微调 |
| 延迟 | NPU INT8、 gaze prediction |
| 功耗 | IR 占空、算法上 NPU |
| 精度 | 商用约 0.5–1.5° 视觉角 |

## 九、面试常问题

**Q：XR 眼动为什么用红外不用 RGB？**  
NIR 下瞳孔对比度好，不受显示内容干扰。

**Q：Glint 是什么？**  
红外 LED 在角膜上的镜面反射，用于几何约束与标定。

**Q：为什么要用户标定？**  
视觉轴个体差异、头显佩戴位置不同，需拟合个性化映射。

**Q：Foveated Rendering 为什么需要 gaze prediction？**  
渲染 pipeline 有延迟，需预测未来注视点，否则降分辨率区域滞后。
