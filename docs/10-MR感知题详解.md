# 10 - MR / 感知题详解

对应 [02-PICO面试题库.md](./02-PICO面试题库.md)「MR / 感知方向」部分。MR（Mixed Reality）要求将**真实世界感知**与**虚拟渲染**在时空、几何、外观上精确融合。

> **GitHub 阅读：** 公式使用 `$...$` / `$$...$$` 格式。  
> 相关文档：[05-系统设计题与模拟面试.md](./05-系统设计题与模拟面试.md)（See-through 系统设计）、[08-影像ISP题详解.md](./08-影像ISP题详解.md)（画质增强）

---

## 总览：MR 感知技术栈

```
真实世界
    ↓ 相机感知
几何理解（深度 / SLAM / 重建）
    ↓
语义理解（分割 / 检测 / VLM）
    ↓
虚实融合（对齐 + 遮挡 + 光照一致）
    ↓
显示（See-through / 合成渲染）
```

**PICO MR 核心挑战：** 低延迟透视、几何准确、长期稳定、端侧算力受限。

## 一、See-through 透视（Video See-Through, VST）

### Q1：相机 → 畸变校正 → 深度 → 虚实对齐 → 增强 → 显示，各环节做什么？

#### 1.1 什么是 See-through？

用户戴上 MR 头显，通过**头显前置相机**采集现实画面，经处理后显示在屏幕上，并叠加虚拟物体 — 形成「透视」效果。

```
VST（视频透视）：相机拍现实 → 屏幕显示（本讲重点）
OST（光学透视）：半透明镜片直接看现实 + 投影虚拟（Hololens 类）
```

PICO 4 Ultra 等 MR 设备主要走 **VST** 路线。

#### 1.2 全 Pipeline 详解

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 相机采集  │ → │ 畸变校正  │ → │ 深度估计  │ → │ 虚实对齐  │ → │ 画质增强  │ → │ 显示合成  │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
   2–3 ms         2–3 ms         5–8 ms         3–5 ms         4–6 ms         3–5 ms
                                                          总计目标：18–25 ms
```

---

**Stage 1：相机采集**

| 要点 | 说明 |
|------|------|
| 多相机 | 双目/广角，覆盖 MR FOV |
| 同步 | 硬件 FSIN，时间戳对齐（见 [08-影像ISP题详解.md](./08-影像ISP题详解.md)） |
| 曝光 | 与显示亮度协调，避免过曝窗外、欠曝室内 |
| Rolling shutter | 快速转头时几何畸变，需 IMU 辅助 |

---

**Stage 2：畸变校正（Undistortion）**

相机镜头有径向/切向畸变，需 remap 到理想针孔模型：

$$
x_{dist} = x(1 + k_1 r^2 + k_2 r^4) + \cdots
$$

- 用标定得到的 $K$ 和畸变系数  
- 预计算 **LUT** + 双线性插值，实时查表  
- **显示端**也有透镜畸变，需分别校正后在统一坐标系融合  

**面试点：** 相机畸变校正后的图像，才能与 SLAM/深度网络训练分布一致。

---

**Stage 3：深度估计**

为虚实融合提供**场景几何**：

| 方法 | 说明 |
|------|------|
| 双目立体匹配 | 三角测量，精度好，需标定基线 |
| ToF / 结构光 | 主动深度，暗光可能差 |
| 单目深度网络 | 一个相机即可，相对深度需 scale |
| SLAM 稀疏深度 | 特征点深度 + 稠密化 |

深度图 $D(u,v)$：每个像素到相机的距离。

**用途：**
- 虚拟物体**遮挡**关系（虚拟桌子挡住真实地板）  
- **重投影**到显示坐标  
- **碰撞检测**  

---

**Stage 4：虚实对齐（Registration / Alignment）**

将虚拟内容精确「贴」在现实世界上。

**坐标系链：**

$$
T_{display}^{virtual} = T_{display}^{head} \cdot T_{head}^{world} \cdot T_{world}^{virtual}
$$

| 对齐类型 | 内容 |
|----------|------|
| **几何对齐** | 虚拟物体位置/尺度与真实场景一致（靠 SLAM + 深度） |
| **时间对齐** | 相机帧与头部运动补偿，减 MTP 延迟 |
| **色彩对齐** | 虚拟物体光照/色调匹配真实（见 Stage 5） |
| **遮挡对齐** | 深度比较决定前后景 |

**重投影（把相机图像 warp 到显示视角）：**

若头部位移较小，可用深度将源视角像素投影到目标视角：

$$
\begin{bmatrix} u' \\ v' \\ 1 \end{bmatrix}
\sim K' \left[ R | t \right] D(u,v) K^{-1} \begin{bmatrix} u \\ v \\ 1 \end{bmatrix}
$$

快速转头时需 **reprojection + 空洞填充** 或 **3D warp**。

---

**Stage 5：画质增强**

透视画面常需 ISP/计算摄影处理（详见 [08-影像ISP题详解.md](./08-影像ISP题详解.md)）：

| 处理 | 目的 |
|------|------|
| 去噪 | 低光颗粒感 |
| HDR / tone mapping | 高动态范围场景 |
| 色彩校正 | 与虚拟内容色域一致（sRGB/P3） |
| 去模糊 | 运动模糊 |
| 锐化 | 提升清晰度（控制强度） |

**权衡：** 增强越强延迟越高；MR 优先保 **延迟 < 画质极致**。

---

**Stage 6：显示合成**

```
增强后的现实世界纹理（全屏或部分）
        +
虚拟渲染帧（带 alpha / 深度）
        ↓
Compositing（按深度排序混合）
        ↓
显示畸变校正 → 屏幕
```

**深度合成规则：**

```
if z_virtual < z_real(pixel):
    显示虚拟像素（虚拟遮挡现实）
else:
    显示现实像素
```

**Latency 关键：** 显示帧应使用**最新**相机帧或做 1 帧预测外推，避免「游泳感」。

#### 1.3 评测指标

| 指标 | 说明 |
|------|------|
| MTP 延迟 | 头部运动到画面更新 |
| 几何误差 | 虚拟物体与真实锚点偏差（cm） |
| 色彩一致性 | MOS 主观评分 |
| 晕动症 | 舒适度量表 |
| 遮挡正确率 | 虚实前后景是否正确 |

#### 1.4 失败 Fallback

| 场景 | 策略 |
|------|------|
| 深度失效 | 固定平面假设（地面） |
| 过暗 | 增强 + 降帧率 |
| 快速运动 | 降增强，优先延迟 |
| SLAM 丢失 | 冻结虚拟锚点，提示用户 |

**完整系统设计答案：** 见 [05-系统设计题与模拟面试.md](./05-系统设计题与模拟面试.md) 第一节。

## 二、场景重建（NeRF / 3DGS / SLAM）

### Q2：NeRF、3DGS、SLAM 如何选型？

#### 2.1 三种技术定位

| 技术 | 输出 | 核心思想 | 实时性 |
|------|------|----------|--------|
| **SLAM** | 稀疏/半稠密地图 + 位姿 | 增量估计相机轨迹与环境 | 实时 ✓ |
| **NeRF** | 隐式辐射场 | MLP 拟合 $(x,y,z,\theta,\phi)\to(color, density)$ | 训练慢，推理慢 |
| **3DGS** | 显式高斯点云 | 百万 3D Gaussian + 可微光栅化 | 训练中等，**渲染快** ✓ |

#### 2.2 SLAM（Simultaneous Localization and Mapping）

**同时定位与建图：** 未知环境中估计相机位姿 $T_t$，并构建地图。

**视觉 SLAM 框架（如 ORB-SLAM）：**

```
Tracking（前端）→ 每帧位姿，跟踪特征点
Local Mapping  → 局部 BA 优化
Loop Closing   → 检测回环，消除累积漂移
```

**输出：**
- 相机轨迹（6DoF 时间序列）  
- 稀疏 3D 点云 / 关键帧  
- 可选稠密重建（RGB-D、MVS）  

**MR 用途：**
- 头显定位（最核心）  
- 虚拟物体锚定在世界坐标  
- 碰撞平面估计  

**优点：** 成熟、实时、闭环消漂移  
**缺点：** 纹理少区域失败；稠密几何需额外模块  

---

#### 2.3 NeRF（Neural Radiance Fields）

**隐式表示：** 用神经网络存储场景，查询任意 3D 点的颜色和密度：

$$
(c, \sigma) = \text{MLP}(\mathbf{x}, \mathbf{d})
$$

**体渲染沿射线积分：**

$$
C(\mathbf{r}) = \int_{t_n}^{t_f} T(t) \cdot \sigma(\mathbf{r}(t)) \cdot c(\mathbf{r}(t), \mathbf{d}) \, dt
$$

$$
T(t) = \exp\left(-\int_{t_n}^{t} \sigma(\mathbf{r}(s)) \, ds\right)
$$

**优点：** 新视角合成质量极高，细节丰富  
**缺点：** 训练慢（每场景数小时），推理需大量采样，**难实时**  

**MR 适用：** 离线场景扫描、数字孪生、预录环境展示；不适合每帧在线 MR 透视。

---

#### 2.4 3D Gaussian Splatting（3DGS）

**显式表示：** 场景 = 大量 3D 高斯椭球，各有位置、协方差、颜色、不透明度。

**渲染：** 将 3D 高斯投影到 2D，按深度排序 alpha blending — **可微光栅化**，GPU 友好。

**优点：**
- 渲染速度 **30–100+ FPS**（桌面 GPU）  
- 质量接近 NeRF  
- 显式几何，可编辑  

**缺点：**
- 经典版仍需 **per-scene 训练**（分钟级）  
- 存储较大（百万高斯）  
- 动态场景、端侧部署仍在研究  

**MR 适用：** 房间扫描后快速预览；与 SLAM 结合做稠密地图；PICO MR 组研究方向。

##### 2.4.1 每个 scene 都要重新训练吗？

| 路线 | 是否 per-scene 重训 | 说明 |
|------|---------------------|------|
| **经典 3DGS** | **是** | 每个场景单独优化一套高斯参数 |
| **Feed-forward 3DGS**（pixelSplat、MVSplat） | 否 | 大模型前向推理，秒级出高斯 |
| **SLAM + 增量 3DGS** | 增量更新 | 新帧增删高斯，不全量重训 |
| **同场景更新** | 微调 | 家具变动后在原高斯上继续优化 |
| **MR 产品常见** | 离线训好复用 | 扫房间 → 云端训练 → 设备加载 |

**核心：** 经典 3DGS 不是「一个网络通吃所有房间」，而是把**该场景**直接编码成百万高斯；换场景通常要重新训练（或走 feed-forward 新路线）。

```
场景 A（客厅）→ 训练 → 高斯集合 GA（不能用于卧室）
场景 B（卧室）→ 重新训练 → 高斯集合 GB
```

##### 2.4.2 经典 3DGS 训练流程

**输入：**
1. 多视角图像（几十～几百张）  
2. 相机位姿 $R, t$ 和内参 $K$（通常 **COLMAP** SfM 估计）

**初始化：** COLMAP 稀疏点云 → 每个点变成一个 3D Gaussian 种子。

**每个 Gaussian 优化参数：**

| 参数 | 含义 |
|------|------|
| 位置 $\mu$ | 3D 中心 |
| 协方差 $\Sigma$ | 椭球形状（常分解为 scale + 四元数旋转） |
| 不透明度 $\alpha$ | 透明度 |
| 颜色 SH | 球谐系数，视角相关颜色 |

**训练循环：**

```python
# 初始化：COLMAP 点云 → Gaussians
for iteration in range(30_000):          # 通常 7k～30k iter
    cam = sample_random_training_camera()
    image_pred = gaussian_rasterize(gaussians, cam)   # 可微光栅化
    loss = L1(image_pred, cam.image) + λ * D_SSIM(...)
    loss.backward()
    optimizer.step()                       # Adam，分组学习率

    if iteration in densify_range:
        densify_and_prune(gaussians)     # clone / split / prune
```

**损失：**

$$
\mathcal{L} = (1-\lambda) \| \hat{I} - I \|_1 + \lambda \cdot \mathrm{D\text{-}SSIM}(\hat{I}, I)
$$

**致密化（Densification）—— 3DGS 关键机制：**

| 操作 | 直觉 |
|------|------|
| **Clone** | 欠重建、梯度大、高斯太小 → 复制 |
| **Split** | 高斯太大、细节不够 → 分裂 |
| **Prune** | 不透明度过低 → 删除 |

从几万 SfM 初值 → 优化到 **几十万～几百万** 高斯。

**资源参考：** 桌面 GPU 约 **5～30 分钟**，显存 **8～24 GB**。训完后推理很快，**新视角**无需再训练；**新场景**需重新训练。

##### 2.4.3 可微光栅化怎么做？

与传统三角形光栅化不同，3DGS 是 **「3D 高斯 → 2D splat → alpha 混合」**。

**① 3D 高斯密度：**

$$
G(\mathbf{x}) = \exp\left(-\frac{1}{2}(\mathbf{x}-\mu)^T \Sigma^{-1} (\mathbf{x}-\mu)\right)
$$

**② 投影到 2D（EWA）：** 3D 协方差经相机雅可比 $J$、旋转 $W$ 映射到图像平面：

$$
\Sigma'_{2D} = J \, W \, \Sigma \, W^T \, J^T
$$

屏幕上的 2D 高斯足迹：

$$
G'(p) = \exp\left(-\frac{1}{2}(p - \mu_{2D})^T \Sigma'^{-1}_{2D} (p - \mu_{2D})\right)
$$

**③ 像素权重：** $w(p) = \alpha \cdot G'(p)$

**④ 按深度排序后 Alpha Blending（front-to-back）：**

$$
C(p) = \sum_{i=1}^{N} c_i \cdot \alpha_i(p) \cdot \prod_{j=1}^{i-1}(1 - \alpha_j(p))
$$

其中 $\alpha_i(p) = o_i \cdot G'_i(p)$，$o_i$ 为该高斯不透明度。

**⑤ GPU 实现要点：**
- 屏幕分 **tile**（如 16×16），每高斯只影响覆盖的 tile  
- Tile 内按深度 **radix sort**  
- CUDA 并行 + 保存中间量 → **可反传**梯度到 $\mu, \Sigma, \alpha, SH$  
- 开源：**diff-gaussian-rasterization**

**与 NeRF 对比：**

| | NeRF | 3DGS 光栅化 |
|---|------|-------------|
| 渲染 | 射线采样 + MLP 查询 + 体积分 | 高斯投影 + 排序 alpha blend |
| 速度 | 慢 | 快（30–100+ FPS） |
| 训练 | 每场景优化 MLP | 每场景优化高斯 + 致密化 |

```
离线：用户扫房间 → 多视角图 + COLMAP → 云端训 3DGS → 下发资产
在线：头显 SLAM 定位 + 预加载 3DGS 与真实世界对齐显示
不是：每次开机在头显里现场训 30 分钟
```

#### 2.5 选型决策树（面试口述）

```
需要实时头显定位（每帧）？
  └─ YES → SLAM / VIO（必选基建）

需要在线稠密新视角合成？
  ├─ 可离线训练该房间 → 3DGS（首选）或 NeRF
  └─ 必须实时建图 → RGB-D fusion / 在线 MVS + SLAM

需要照片级静态场景资产？
  └─ 3DGS > NeRF（速度）> MVS 点云

  └─ SLAM 稀疏 + 轻量深度；3DGS/NeRF 放云端或预处理
```

#### 2.6 组合架构（工业界常见）

```
SLAM 提供实时 6DoF 位姿
    +
深度网络提供稠密 depth
    +
用户扫描关键帧 → 云端 3DGS 重建 → 下载资产
    +
MR 运行时：SLAM 定位 + 预加载 3DGS 场景 + 实时 VST
```

## 三、深度估计（Depth Estimation）

### Q3：单目 vs 双目？自监督方法有哪些？

#### 3.1 双目深度（Stereo）

**原理：极线几何 + 三角测量**

同一 3D 点在两相机像平面投影：

$$
d = \frac{f \cdot B}{disparity}
$$

- $f$：焦距  
- $B$：基线（两相机间距）  
- $disparity$：视差（水平像素差）  

**流程：**

```
左右图像（已校正 epipolar）→ 立体匹配（SGBM / RAFT-Stereo / 轻量 CNN）
    → 视差图 → 深度图
```

| 优点 | 缺点 |
|------|------|
| 尺度绝对、精度高 | 需双相机标定与同步 |
| 物理可解释 | 无纹理区域匹配失败 |
| 近处精度高 | 远处精度随距离下降 |

**PICO MR：** 头显前置双目是自然配置，立体深度是首选 baseline。

---

#### 3.2 单目深度（Monocular）

**问题：** 从单张 RGB 估计深度，本质病态 — 多种 3D 解释对应同一 2D 图像。

**尺度模糊：** 网络常输出**相对深度**或**仿射不变深度**，需 SLAM/IMU/已知物体尺度对齐。

**代表方法：**

| 方法 | 特点 |
|------|------|
| **MiDaS** | 大规模混合数据训练，零样本泛化强 |
| **Depth Anything** | 基础模型 + 微调，质量高 |
| **ZoeDepth** | 度量深度（metric depth） |
| **轻量 MobileNet 头** | 端侧实时 |

**优点：** 单相机即可，硬件简单  
**缺点：** 绝对尺度需额外信息；边缘可能糊  

**选型：**
- 有双目 → 双目为主，单目补空洞  
- 仅单目 → 深度网络 + VIO scale  

---

#### 3.3 自监督深度估计

**动机：** 深度标注昂贵（LiDAR、结构光），利用**立体视频**或**单目视频**的几何约束自监督。

**经典：Monodepth2（Godard et al.）**

损失组成：

| 损失 | 作用 |
|------|------|
| **Photometric loss** | 用预测深度将一帧 warp 到另一帧，与真实图像比重建误差 |
| **Smoothness loss** | 深度在纹理弱区域平滑 |
| **Occlusion mask** | 排除遮挡区域 warp |
| **Stereo consistency** | 左右目深度一致（若有双目） |

**Photometric 重建：**

$$
\mathcal{L}_{photo} = \sum_{p} \| I_t(p) - \hat{I}_{t'}(p) \|
$$

$\hat{I}_{t'}$ 由 $D_t$ 和相对位姿 $T_{t\to t'}$ 将 $I_{t'}$ warp 得到。

**其他自监督：**
- **Depth from video + SLAM pose**  
- **Scale-aware self-supervised（ManyDepth）**  
- **Foundation model 蒸馏（Depth Anything v2）**  

**面试要点：**
- 自监督依赖**相机运动**和**场景静态**假设  
- 动态物体是主要误差源 → 需 mask  
- 与 SLAM 联合可解 metric scale  

---

#### 3.4 MR 深度方案对比

| 方案 | 延迟 | 精度 | 端侧可行性 |
|------|------|------|------------|
| 双目 SGM | 低 | 中高 | ✓ |
| RAFT-Stereo INT8 | 中 | 高 | ✓（需优化） |
| 单目 MiDaS 小模型 | 低 | 中 | ✓ |
| ToF | 极低 | 中（距离有限） | ✓ 若有硬件 |

**融合：** 双目 + 单目网络 + 时序滤波（Kalman on depth map）。

## 四、环境理解（Semantic Segmentation for MR）

### Q4：语义分割如何辅助 MR 交互？

#### 4.1 什么是环境理解？

不仅知道「有多深」，还知道「是什么」— 对像素或区域赋予语义类别。

| 层级 | 输出 | 示例 |
|------|------|------|
| **语义分割** | 每像素类别 | 墙、地板、桌子、人 |
| **实例分割** | 每物体实例 | 桌子1、椅子2 |
| **全景分割** | 语义 + 实例 | Panoptic FPN |
| **3D 语义** | 点云/voxel 带标签 | 房间理解 |

#### 4.2 MR 交互用途

| 应用 | 说明 |
|------|------|
| **遮挡与碰撞** | 虚拟球落在「桌子」语义区域，不穿透 |
| **平面检测** | 桌面/墙面作为虚拟物体放置面 |
| **人物处理** | 分割出人，虚拟内容不遮挡真人脸（社交） |
| **场景自适应** | 办公室 vs 客厅不同 UI 布局 |
| **交互锚点** | 「放到沙发上」需识别沙发 |
| **安全** | 检测楼梯、窗户，虚拟边界警告 |
| **光照估计** | 结合分割区域估计环境光方向 |

#### 4.3 典型 Pipeline

```
相机图像 → 轻量分割网络（DeepLabV3+ / SegFormer-T / YOLOv8-seg）
    → 语义 mask（H/4 × W/4 上采样）
    → 与深度图融合 → 3D 语义点云（可选）
    → 交互引擎 / 物理引擎
```

**端侧模型：**

| 模型 | 特点 |
|------|------|
| **BiSeNet / STDC** | 极快，适合实时 |
| **SegFormer-B0** | Transformer，精度好 |
| **YOLOv8-seg** | 检测+分割统一 |

**延迟预算：** 5–15 ms（INT8，512×512 输入）

#### 4.4 与 SLAM / 深度结合

```
2D 语义 mask + 深度 D(u,v)
    → 反投影到 3D
    → 语义点云 / 语义体素
    → 持久化房间 mesh（慢更新）
```

**时序一致性：** 单帧分割抖动 → 用多帧投票或 CRFF / 时序网络平滑。

#### 4.5 与 VLM 的关系

| 能力 | 传统分割 | VLM |
|------|----------|-----|
| 固定类别（80类 COCO） | ✓ 快 | 过度 |
| 开放词汇（「红色马克杯」） | ✗ | ✓ |
| 实时性 | ✓ 端侧 | 通常云端 |
| MR 策略 | 端侧分割做基础交互 | 复杂 query 走云端 |

**混合：** 端侧分割提供地板/墙/桌；用户语音「把模型放到键盘旁」触发 VLM + 检测。

#### 4.6 评测

| 指标 | 说明 |
|------|------|
| mIoU | 分割精度 |
| 端到端延迟 | 分割 + 渲染 |
| 交互成功率 | 虚拟物体是否正确落桌 |
| 时序稳定性 | mask 边界抖动程度 |

## 六、速记表（考前 5 分钟）

| 主题 | 一句话 |
|------|--------|
| VST | 相机拍现实 → 处理 → 屏显 + 虚拟叠加 |
| 透视 pipeline | 采集→畸变→深度→对齐→增强→合成 |
| 虚实对齐 | 统一坐标系 + 深度遮挡 + 色彩一致 |
| SLAM | 实时定位建图，MR 基建 |
| NeRF | 隐式高质量，慢，离线，每场景训 MLP |
| 3DGS | 显式高斯，可微光栅化，渲染快；经典版每场景训，feed-forward 可泛化 |
| 3DGS 训练 | COLMAP 位姿 → 初始化 → L1+D-SSIM → densify/prune |
| 3DGS 光栅化 | 3D→2D 高斯投影 + 深度排序 alpha blend |
| 双目深度 | 视差→深度，尺度准，需标定 |
| 单目深度 | 病态，需 scale，端侧轻量 |
| 自监督 | 光度重建 + 平滑 + 遮挡 mask |
| 语义分割 | 平面/物体/人，支撑放置与交互 |
| 端云 | 分割深度在端，开放语义在云 |
